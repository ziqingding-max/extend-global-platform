# 回归测试检查表 (Regression Checklist)

本检查表用于每次生产环境发版前的最终验证。所有标记为 **[P0]** 的项目必须 100% 通过，否则**禁止发版**。标记为 **[P1]** 的项目如果失败，需要评估影响范围后决定是否阻断发版。

> **使用方法**：QA 负责人在发版前复制本文件，逐项勾选并签名。完成后归档至 `docs/07_测试/历史回归记录/` 目录。

---

## 1. 认证与权限 (Auth & RBAC)

| 优先级 | 检查项 | 通过 | 备注 |
| :---: | :--- | :---: | :--- |
| P0 | Admin 账号可以正常登录 Admin Portal | ☐ | |
| P0 | CP Admin 账号登录后，只能看到归属于自己 CP 的客户和员工数据 | ☐ | |
| P0 | Client Admin 账号登录后，只能看到归属于自己客户的员工数据 | ☐ | |
| P0 | Worker 账号登录后，只能看到自己的个人档案和工资单 | ☐ | |
| P0 | 使用 CP_A 的 Token 尝试请求 CP_B 的数据，返回 403/404 | ☐ | |
| P1 | 密码重置邮件能正常发送并完成重置流程 | ☐ | |
