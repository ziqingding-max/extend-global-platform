# 净额法 P&L 报表 (Net-basis P&L Reporting)

在 Extend Global (EG) 的财务体系中，如何准确衡量平台的真实收入是一个关键的合规与商业分析命题。为了遵循 GAAP/IFRS 会计准则，并剔除庞大但不产生利润的代收代付资金，系统采用了“净额法（Net-basis）”来构建损益表（P&L）。

本文档详细解释净额法的核心概念及其在系统报表中的实现逻辑。

## 1. 总额法 (Gross-basis) vs. 净额法 (Net-basis)

在 EOR 业务中，平台向客户收取的总金额（Gross Billing）通常非常庞大，因为其中包含了员工的全部薪水、税费和社保。

### 1.1 总额法的误区
如果采用总额法记账：
* **收入 (Revenue)** = 客户支付的发票总额（例如 $10,500）
* **成本 (COGS)** = 支付给员工和政府的金额（例如 $10,000）
* **毛利 (Gross Profit)** = $500

这种做法虽然在数学上成立，但在会计准则（如 IFRS 15 / ASC 606）下，**平台在此交易中扮演的是“代理人（Agent）”而非“主要责任人（Principal）”**。将不属于平台的资金计入收入，会严重虚增公司的营收规模，误导投资者与管理层。

### 1.2 净额法的合规性
采用净额法记账：
* **代收代付款 (Pass-through Funds)** = $10,000（计入资产负债表的应付账款/代收代付科目，不计入利润表）
* **净收入 (Net Revenue)** = $500（仅将平台实际赚取的服务费与汇差计入利润表顶行）
* **成本 (COGS)** = 平台自身的运营成本（如支付给底层 Vendor 的系统使用费、银行转账手续费等）

净额法能够真实反映平台的创收能力，是 EG 财务系统的核心核算原则。

## 2. 净额法 P&L 报表结构

系统中的 `Net P&L Report` 模块（由 `netPnlRouter` 提供数据）严格按照净额法结构生成报表。其标准层级如下：

### 2.1 顶行：净收入 (Net Revenue)
这是报表的最顶层，代表平台真正赚取的钱。它由以下几个部分组成：
1. **Management Fees (管理服务费)**：通过定价引擎计算出的按月/按人头收取的服务费。
2. **FX Markup Revenue (汇差利润)**：通过 FX 剥离引擎计算出的，因对客汇率加价而产生的隐性利润。
3. **Setup / One-time Fees (一次性开户/实施费)**：客户入职时收取的一次性费用。
4. **Deposit Interest (保证金利息)**：*(可选)* 冻结钱包中庞大保证金在银行产生的利息收入。

### 2.2 减项：直接销售成本 (Cost of Sales / COGS)
在净额法下，COGS 不包含员工的薪水，只包含为促成这笔净收入而发生的直接成本：
1. **Vendor Processing Fees (供应商处理费)**：支付给当地落地 Vendor 的按人头计算的手续费（注意：不是代发的薪水，纯粹是 Vendor 收取的手续费）。
2. **Payment Gateway Fees (支付网关手续费)**：通过 Stripe, Airwallex 等渠道收付款时产生的手续费。
3. **CP Commission / Margin (渠道分成)**：在 B2B2B 模式下，如果以终端客户支付的 Layer 2 金额作为起点，必须减去留存在 CP 钱包中的利润差额（Layer 2 - Layer 1）。

### 2.3 底行：毛利 (Gross Profit)
`Gross Profit = Net Revenue - COGS`
这是平台在扣除所有直接交易成本后，可用于支付内部员工工资、研发和营销费用的资金。

## 3. 多维度分析能力

为了支持精细化的商业决策，系统的 P&L 报表引擎支持在多个维度上进行数据切片与下钻分析（Drill-down）。

### 3.1 按渠道伙伴 (By Channel Partner)
分析哪些 CP 带来了最高的净收入。由于不同的 CP 可能有不同的底价协议（Layer 1 Pricing），通过这个维度的分析，EG 可以评估每个渠道的盈利能力与 ROI。

### 3.2 按收入类型 (By Revenue Stream)
分析净收入的构成比例。例如，发现某个国家的业务中，`FX Markup Revenue` 占比高达 60%，而 `Management Fees` 仅占 40%，这提示管理层该国家的盈利高度依赖于汇率波动，可能存在一定的金融风险。

### 3.3 按国家/地区 (By Country/Region)
分析不同国家的利润率。由于各国的 Vendor 成本差异巨大，某些国家可能收入很高但 Vendor 抽成也高，导致最终的毛利率（Gross Margin）反而不如其他国家。

## 4. 数据一致性保障

为了确保 P&L 报表的绝对准确，系统在底层设计上做了严格的校验：
* **不可变账单 (Immutable Invoices)**：一旦发票状态变为 `Published`，其关联的费用明细项即被锁定，任何修改必须通过生成红字发票（Credit Note）来进行。
* **双重校验**：P&L 引擎在汇总数据时，会交叉比对 `invoices` 表中的汇总金额与 `invoiceItems` 表中的明细累加值，确保在剥离 Pass-through 成本时不会出现舍入误差。

通过实施严格的净额法 P&L 报表，Extend Global 不仅满足了严苛的审计与合规要求，更为管理层提供了一面清晰透视平台真实盈利能力的“镜子”。
