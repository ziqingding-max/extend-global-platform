# 四方角色与流转机制

在 Extend Global (EG) 的 B2B2B 商业模型中，业务链路贯穿了四个核心实体。本文档详细阐述这四方角色在系统中的交互流程与责任边界，这是理解系统复杂权限与数据隔离机制的前提。

## 1. 核心流转全景

系统的核心业务流转可以概括为“一条主线、两个闭环”。

* **一条主线**：终端客户 (Client) 提出用工需求，员工 (Worker) 提供劳动，平台完成跨国发薪与合规。
* **两个闭环**：
  * **服务交付闭环**：EG 提供底层基础设施 → CP 包装品牌对外销售 → Client 获得全球雇佣能力。
  * **资金结算闭环**：Client 向 CP 支付全额账单 → CP 向 EG 支付底层成本 → EG 向当地 Vendor 或直接向 Worker 发放薪酬。

## 2. 角色职责与边界定义

为了保证系统的稳定运行与合规性，四方角色在系统中的职责被严格定义，且相互之间存在明确的可见性边界。

### 2.1 Extend Global (EG) - 基础设施提供方
作为系统的所有者，EG 在整个流转中扮演“幕后英雄”的角色。EG 负责维护全球各国的法律实体网络，确保各地的劳动法与税务合规。在系统中，EG 运营人员通过 Admin Portal 进行全局管理，设定针对不同 CP 的底价策略，并处理最终的跨国资金下发。终端客户在整个服务周期内，完全感知不到 EG 的存在。

### 2.2 Channel Partner (CP) - 渠道与品牌方
CP 是连接 EG 与终端客户的桥梁。CP 利用 EG 提供的系统能力，通过配置自己的域名、Logo 和主色调，构建出一个专属的“白标门户”。CP 的核心职责是获客与客户关系维护。在系统中，CP 管理员可以为旗下不同的客户设置差异化的加价策略，并负责向客户催收账单。CP 必须在其系统钱包中保持充足的余额，以支付 EG 的底层服务费。

### 2.3 Client - 终端用工企业
Client 是实际产生雇佣需求并为服务买单的实体。Client HR 或财务人员登录由 CP 提供的白标门户，发起员工入职邀请，审批日常的请假与报销申请。每月初，Client 会收到由 CP 开具的包含服务费加价的账单，并直接将款项支付至 CP 的线下银行账户。Client 的数据被严格限制在其所属的 CP 范围内。

### 2.4 Worker - 实际劳动者
Worker 分为全职员工 (EOR) 与独立承包商 (AOR)。他们通过专属的员工门户完成自助入职、签署合同、提交报销与查看工资单。与 Client 类似，Worker 看到的也是 CP 的品牌界面。全职员工在法律上与 EG 在当地的实体（或 Vendor）签署劳动合同，但在日常工作中，他们接受 Client 的业务指令。

## 3. 典型业务场景流转

以下通过两个最核心的业务场景，展示四方角色是如何协同工作的。

### 3.1 员工入职流转 (Onboarding Flow)

员工入职是整个服务周期的起点，涉及多方的信息确认与合规审查。

1. **需求发起**：Client HR 在 App Portal 中填写拟雇佣员工的基本信息（职位、薪资、国家），系统根据该国家的要求生成初步的成本预估。
2. **邀请发送**：Client 确认后，系统以 CP 的名义向候选人发送包含入职链接的邀请邮件。
3. **自助入职**：Worker 点击链接进入白标员工门户，填写详细的个人信息、紧急联系人，并上传身份证件与银行账户凭证。
4. **合规审查**：EG 运营团队在 Admin Portal 中收到入职申请，进行 AML（反洗钱）与当地劳动法合规审查。
5. **合同签署**：审查通过后，系统生成标准的当地劳动合同（或服务协议）。Worker 在线签署，随后 EG 实体代表也完成签署。
6. **入职生效**：合同生效，Worker 状态变更为 Active，正式开始提供劳动。

### 3.2 每月发薪与结算流转 (Payroll & Billing Flow)

每月的发薪结算是系统最复杂、也是最核心的业务流转。

1. **薪酬计算**：月末，系统根据 Worker 的基础薪资、当月获批的请假扣款与报销金额，生成初步的 Payroll Run 数据。
2. **双层发票生成**：系统基于底价与加价规则，同时生成两份账单：
   * Layer 1 Invoice：EG 开给 CP 的底价账单。
   * Layer 2 Invoice：CP 开给 Client 的加价账单。
3. **客户付款**：Client 在门户中查看 Layer 2 账单，并将款项线下汇入 CP 的银行账户。
4. **CP 资金流转**：
   * CP 在确认收到 Client 款项后，在系统中将 Layer 2 账单标记为“已支付”。
   * 随后，CP 进入系统钱包，手动点击“支付”，从其钱包余额中扣除 Layer 1 账单的金额给 EG。
5. **资金下发**：EG 在确认收到 CP 的底层成本款项后，通过全球支付网络，将税后净薪资发放至 Worker 的个人银行账户，并将相应的税费缴纳至当地政府机构。

通过这种严密的流转机制，Extend Global 平台成功实现了资金流、信息流与合规责任的有效隔离与高效协同，支撑起复杂的 B2B2B 商业生态。
